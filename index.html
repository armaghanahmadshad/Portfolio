01 / about

Still early. Already building.

I'm currently a 2nd-semester BS Information Technology student at Muhammad Nawaz Sharif University of Engineering & Technology (MNS UET), Multan — so this portfolio is honest about where I am: a student building real, working projects while I learn cloud and DevOps properly, not a resume padded with invented job history.

As a Microsoft Learn Student Ambassador, I run technical sessions and workshops for the student community, and I also teach part-time at The Knowledge School. Between both, I've gotten comfortable explaining infrastructure concepts to people who've never touched a terminal — which turns out to be excellent practice for writing clear runbooks.

My focus right now: Git workflows, Python tooling, and the first layer of the DevOps stack — CI/CD, infrastructure as code, and containers — with cloud computing as the direction I'm growing into.

04 / projects

Learning-in-public builds

Three practice projects that map to the core DevOps stack. Each repo README documents what broke and what I'd do differently — that's most of the value.

CI / CD

Automated pipeline for a Flask API

A small Python/Flask service with a GitHub Actions workflow that lints, runs pytest, builds a Docker image, and deploys to Render on every push to main. Built to understand exactly what a "green pipeline" is actually checking.

  • Python
  • Flask
  • GitHub Actions
  • Pytest
  • Docker

INFRASTRUCTURE AS CODE

Terraform sandbox on Azure

Reproducible infrastructure for a single Linux VM and networking, defined entirely in Terraform. Focus was on state management and writing a plan a reviewer could actually trust before typing "apply" on cloud spend.

  • Azure

Monitored multi-container app

A web + worker app orchestrated with Docker Compose, with Prometheus scraping metrics and a Grafana dashboard on top. Built specifically to practice reading dashboards under simulated load rather than only shipping the containers.
